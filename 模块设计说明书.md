## 3. 模块详解

### 3.1 认证模块（auth/）

| 类/函数 | 文件 | 职责 |
|---------|------|------|
| `SignInPage` | `auth/signin.cpp` | 登录界面：卡片布局、密码掩码+眼睛切换、网络请求 |
| `SignUpPage` | `auth/signup.cpp` | 注册界面：用户名/密码/班级/角色、密码复杂度本地校验 |
| `login()` | `server/modelA/login.py` | 登录 API：账号锁定（5次/30分）、SHA-256 密码比对 |
| `register()` | `server/modelA/register.py` | 注册 API：用户名唯一性、SHA-256 哈希存储 |

**密码流程：**

```
客户端：明文 → 本地复杂度校验 → SHA-256 → JSON 传输
服务端：直接存储/比对哈希（无 salt）
```

**安全机制：**
- 连续 5 次登录失败 → 锁定账号 30 分钟
- 密码复杂度本地校验（6~18位、大小写字母、数字、特殊字符）
- 服务端不存储明文密码

### 3.2 学生主页（student/home/）

| 组件 | 类名 | 职责 |
|------|------|------|
| 主窗口 | `StudentMainWindow` | QStackedWidget：主页(0) / 编辑资料(1) |
| 左面板 | `StudentLeftPanel` | 蓝色主题圆形头像、用户名/班级/角色、头像自动加载 |
| 内容区 | `StudentContentArea` | 5个Tab + 卡片网格 + 分页动画 |
| 侧边栏 | `StudentSidebar` | 4项导航（个人中心/视频区/论坛/设置） |

**5个数据 Tab：**

| Tab | API 端点 | 数据源 |
|-----|---------|--------|
| 最近播放 | `GET /api/user/history?days=7` | view_history 表（7天内） |
| 最近下载 | `GET /api/user/downloads` | download_history 表 |
| 最近上传 | `GET /api/user/uploads` | courses + resources 表 |
| 浏览历史 | `GET /api/user/history` | view_history 表（全部） |
| 我的收藏 | `GET /api/user/favorites` | favorites 表 |

**卡片网格：** 每页 3列×2行=6张，含 QPainter 绘制缩略图色块、QGraphicsDropShadowEffect 阴影、QPropertyAnimation 滑动翻页（350ms OutCubic）。

### 3.3 教师主页（teacher/home/）

| 组件 | 类名 | 差异（与学生相比） |
|------|------|-------------------|
| 主窗口 | `TeacherMainWindow` | 绿色主题 #4A7C59 |
| 左面板 | `TeacherLeftPanel` | 绿色头像、欢迎语含教师名 |
| 内容区 | `TeacherContentArea` | 双层 m_viewStack：正常视图(0) / 学生管理占位(1) |
| 侧边栏 | `TeacherSidebar` | 5项导航（含学生管理） |

**当前实现：**
- "最近上传" → `GET /api/user/uploads`（真实数据）
- 学生管理页 → 空白占位（等待后续前端参考）
- Sidebar 导航路由 → 未实现（等待后续前端参考）

### 3.4 编辑个人资料（shared/profile_editor/）

| 组件 | 类名 | 职责 |
|------|------|------|
| 编辑器 | `ProfileEditWidget` | 编辑页主控件，含 backClicked / avatarUpdated 信号 |
| 头像控件 | `EditAvatarWidget` | 150px 圆形头像，支持本地预览 + 服务端上传 |

**功能：**

| 功能 | 客户端逻辑 | 服务端端点 |
|------|-----------|-----------|
| 修改用户名 | 本地格式校验 → POST | `/api/user/change-username` |
| 修改密码 | 本地复杂度校验 → SHA-256 → POST | `/api/user/change-password` |
| 上传头像 | 选图 → 本地预览 → multipart POST | `/api/user/upload-avatar` |
| 加载头像 | GET 图片数据 → QPixmap 显示 | `/api/user/avatar/<username>` |

**信号流：**
- `backClicked` → 关闭编辑页，返回主页
- `avatarUpdated` → 通知主页左面板重新加载头像

### 3.5 公共工具（common/）

| 文件 | 职责 |
|------|------|
| `base.h / base.cpp` | 抽象基类 `Base : QWidget`，定义虚接口 |
| `network_handler.h/.cpp` | HTTP 请求单例，封装 GET/POST/DELETE + JSON 解析 |
| `common.qss` | 全局样式表（苹果风：白色背景、蓝色 #0071E3 强调） |

**NetworkHandler API：**

```cpp
NetworkHandler::instance()->get(url, callback);
NetworkHandler::instance()->post(url, body, callback);
NetworkHandler::instance()->del(url, callback);
NetworkHandler::instance()->manager();  // 底层 QNetworkAccessManager（用于 multipart 上传）
```

## 4. 数据库设计（8张表）

### 4.1 用户表（A）

```sql
CREATE TABLE users (
    user_id     INTEGER PRIMARY KEY AUTOINCREMENT,
    username    TEXT UNIQUE NOT NULL,
    keyword     TEXT NOT NULL,              -- SHA-256 哈希
    class       INTEGER NOT NULL,
    role        TEXT NOT NULL DEFAULT 'student',
    fail_count  INTEGER DEFAULT 0,
    lock_until  INTEGER,
    avatar_path TEXT                        -- 头像文件名（NULL 未设置）
);
```

### 4.2 课程视频表（B）

```sql
CREATE TABLE courses (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    course    TEXT NOT NULL,
    teacher   TEXT NOT NULL,                -- 关联 users.username
    time      TEXT NOT NULL,
    file_path TEXT NOT NULL,
    class     INTEGER NOT NULL
);
```

### 4.3 资源表（C）

```sql
CREATE TABLE resources (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    uploader  TEXT NOT NULL,
    file_path TEXT NOT NULL,
    class     INTEGER NOT NULL,
    time      TEXT NOT NULL,
    course    TEXT NOT NULL
);
```

### 4.4 问答表（D）

```sql
CREATE TABLE questions (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    uploader  TEXT NOT NULL,
    post_id   INTEGER NOT NULL,
    parent_id INTEGER NOT NULL DEFAULT 0,   -- 0=新帖, 非0=回复
    class     INTEGER NOT NULL,
    time      TEXT NOT NULL,
    course    TEXT NOT NULL,
    content   TEXT NOT NULL
);
```

### 4.5 弹幕表（E）

```sql
CREATE TABLE danmaku (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    video_id    INTEGER NOT NULL,
    sender      TEXT NOT NULL,
    content     TEXT NOT NULL CHECK(length(content) <= 50),
    play_time   INTEGER NOT NULL,
    class       INTEGER NOT NULL,
    create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_danmaku_video ON danmaku(video_id, id);
CREATE INDEX idx_danmaku_class ON danmaku(class);
CREATE INDEX idx_danmaku_create ON danmaku(create_time);
```

### 4.6 浏览历史表（F）

```sql
CREATE TABLE view_history (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    username    TEXT NOT NULL,
    video_id    INTEGER NOT NULL,
    video_title TEXT,
    teacher     TEXT,
    class       INTEGER NOT NULL,
    view_time   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_history_user ON view_history(username, view_time);
CREATE INDEX idx_history_class ON view_history(class);
```

### 4.7 下载记录表（G）

```sql
CREATE TABLE download_history (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    username      TEXT NOT NULL,
    file_name     TEXT NOT NULL,
    file_type     TEXT,
    file_size     INTEGER,
    class         INTEGER NOT NULL,
    download_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_download_user ON download_history(username, download_time);
```

### 4.8 收藏夹表（H）

```sql
CREATE TABLE favorites (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    username   TEXT NOT NULL,
    item_type  TEXT NOT NULL CHECK(item_type IN ('video', 'resource')),
    item_id    INTEGER NOT NULL,
    item_title TEXT,
    class      INTEGER NOT NULL,
    added_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_fav_user ON favorites(username, class);
CREATE INDEX idx_fav_item ON favorites(item_type, item_id);
```

## 5. API 接口清单

### 5.1 认证

| 方法 | 路径 | 功能 | 请求体 |
|------|------|------|--------|
| POST | `/api/auth/register` | 注册 | `{username, password(SHA-256), class, role}` |
| POST | `/api/auth/login` | 登录 | `{username, password(SHA-256)}` |

**登录响应：** `{success, role, class, user_id, username, avatar_path}`

### 5.2 课程

| 方法 | 路径 | 功能 | 参数 |
|------|------|------|------|
| GET | `/api/courses` | 课程列表 | `?class=&tag=&teacher=&keyword=` |
| POST | `/api/courses/upload` | 上传视频 | multipart（待实现） |

### 5.3 用户数据

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/user/history` | 浏览历史（`?days=7` 最近播放） |
| POST | `/api/user/history` | 记录浏览 |
| GET | `/api/user/downloads` | 下载记录 |
| POST | `/api/user/downloads` | 记录下载 |
| GET | `/api/user/uploads` | 上传列表（视频+资源） |
| GET | `/api/user/favorites` | 收藏列表 |
| POST | `/api/user/favorites` | 添加收藏 |
| DELETE | `/api/user/favorites/<id>` | 取消收藏 |

### 5.4 个人设置

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/user/change-password` | 修改密码：`{username, old_password, new_password}` |
| POST | `/api/user/change-username` | 修改用户名：`{username, new_username}`（无密码校验） |
| POST | `/api/user/upload-avatar` | 上传头像：multipart `(file + username)` |
| GET | `/api/user/avatar/<username>` | 获取头像文件流 |

### 5.5 弹幕

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/danmaku/init` | 弹幕初始化（最近200条） |
| GET | `/api/danmaku/poll` | 增量轮询 |
| POST | `/api/danmaku/send` | 发送弹幕 |

### 5.6 资源

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/resources` | 资源列表 |
| POST | `/api/resources/upload` | 上传资源（multipart） |
| GET | `/api/questions` | 问答列表 |
| POST | `/api/questions/publish` | 发布问答 |
| GET | `/api/files/<file_id>` | 下载文件 |

## 6. 页面路由与跳转


## 7. 关键设计决策

| 决策 | 说明 |
|------|------|
| 密码传输 | 客户端 SHA-256 哈希后传输，服务端不存明文、无 salt |
| 密码校验 | 复杂度检查在客户端完成，服务端仅检查非空 |
| 用户名修改 | 不校验密码，无级联外键，手动 UPDATE 8 张表 |
| 班级隔离 | 所有查询强制注入 `WHERE class = ?` |
| 账号锁定 | 5 次失败锁定 30 分钟（服务端校验） |
| 头像存储 | 保存至 `uploads/avatars/{user_id}_{timestamp}.{ext}` |
| 头像访问 | 用户名查询 → 返回文件流（GET），不受班级隔离限制 |
| 弹幕轮询 | 客户端 2 秒定时器，GET 增量拉取 |
| 构建系统 | CMake + MinGW，GLOB_RECURSE 自动收集源文件 |
| 样式 | 全局 QSS 苹果风洁白主题 |
